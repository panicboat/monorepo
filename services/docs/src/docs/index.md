# Project Overview

## 1. ビジョンとミッション

### Vision: "店舗"から"個"へ。流動する資産を、確固たる価値へ。

現在の風俗業界のプラットフォームは「店舗」を主語としており、キャストは消耗品として扱われています。
主語を「キャスト個人」に転換し、彼女たちの魅力・信用・ファンとの関係性を**「持ち運び可能な資産（Portable Assets）」**として蓄積するプラットフォームです。

### Mission: "The Premium Promise" (特別な約束体験)

私たちの提供価値は、単なる予約枠の売買（自販機的体験）ではありません。
予約プロセスを**「情緒的な儀式」**へと昇華させ、キャストにはブランドと安心を、ユーザーには高揚感と独占欲を提供します。

## 2. コアコンセプト (The Heart of Service)

本サービスのUI/UXは、全て以下のコンセプトに基づいて設計されています。開発者はこれを理解した上で実装を行う必要があります。

### A. 脱・事務処理 (Ritual over Transaction)

* **Concept:** 予約は「作業」ではなく「誓い」である。
* **UX:**
* キャストは「空き枠」を提示するのではなく、**「招待状（Invitation）」**を送る。
* ユーザーはボタンをタップするのではなく、**「長押し（Long Press）」**することで誓約（予約）を確定させる。
* 確定後は「封蝋（Sealed）」され、安易なキャンセルを許さない心理的拘束力を持たせる。



### B. 生きたポートフォリオ (Living Portfolio)

* **Concept:** カタログスペックではなく、現在の「状態」と「信用」を売る。
* **UX:**
* **No Calendar:** 事務的な空き枠カレンダーは排除する。「他の客の影」を見せないため、そして「交渉」の余地を残すため。
* **Status Signal:** 代わりに「🟢 即レス」「🟡 今夜空きあり」といったリアルタイムなステータスを表示する。
* **Trust Score:** 指名本数ではなく「約束履行率（Promise Rate）」や「来店客のみが書けるレビュー」を最重要指標とする。



### C. 秩序ある聖域 (Sanctuary)

* **Concept:** 無法地帯（ドタキャン・冷やかし）を許さない。
* **UX:**
* No Show（無断キャンセル）を行ったユーザーは、キャストからの報告によりペナルティが付与される。
* この「負の履歴」は全キャストに共有され、アプリ全体の治安を維持する。

## 3. New Joiner 向けガイド (For Contributors)

### 開発の心得

1. **"Webっぽさ"を消せ:** 私たちが作っているのはWebサイトではなく「没入型アプリ」です。ボタンのタップ、画面の切り替わり一つ一つに `Framer Motion` 等で心地よいフィードバックを与えてください。
2. **キャストファースト:** 管理画面や入力フォームは、ネイルをした指でも操作しやすいよう、タップ領域を大きく、入力ステップを極限まで減らしてください。
3. **Code Consistency:** デザインの一貫性を保ってください。黒(#000)・金(#CurrentGold)・赤(#Danger) のカラーパレットを厳守してください。
