# Proposal: Implement Cast Feature

**Goal**: "No Shop, Just Casts" （店舗を持たず、キャストが主役）の理念を実現するため、中核となる `Cast` エンティティと関連サービスを実装する。

## Why
現在、システムはゲストユーザーのみをサポートしている。プラットフォームのビジョンを実現するためには、以下が必要である：
1.  **Cast Registration**: キャストが参加するための専用フロー。
2.  **Cast Profile**: プラットフォームの核心となる資産（「店舗」の代替）。
3.  **Discovery**: ゲストがキャストを見つけられる機能。

本変更は、これらの機能を実現するためのバックエンドおよびフロントエンドの基盤を構築するものである。

## Scope
- **Identity**: GRPC API を更新して `role` (Guest/Cast) をサポートし、フロントエンドの認証UIを実装する。
- **Monolith**: `casts` テーブルを作成し、`CastService` (GRPC) を実装する。**ドメイン分離のため、他スキーマへのFKは設定しない。**
- **Nyx**: キャストデータをポートフォリオページ（検索・一覧）に統合する。
